## 🏗️ Parte 1 e 2: Criação e Nomeação de VLANs

O objetivo aqui é criar as "redes virtuais" que irão segmentar os laboratórios. Essa configuração deve ser feita tanto no **Switch Core** quanto nos **Switches de Acesso**.

### Comandos Úteis

- `vlan <ID>`
    - **Função:** Cria uma VLAN com o número de identificação (ID) que você especificar ou entra no modo de configuração dessa VLAN se ela já existir.
- `name <NOME_DA_VLAN>`
    - **Função:** Atribui um nome descritivo à VLAN que você está configurando. É uma boa prática usar nomes que identifiquem a finalidade da VLAN (ex: `LAB_ENGENHARIA` ou `VLAN10_LAB1`).

### Dica Importante

- A organização é fundamental. Certifique-se de que os IDs e nomes das VLANs sejam consistentes em todos os switches para facilitar o gerenciamento.

### Comando de Verificação

- `show vlan brief`
    - **Função:** Exibe uma tabela com todas as VLANs criadas no switch, seus nomes e quais portas estão associadas a elas. Use este comando para confirmar que suas VLANs foram criadas corretamente.

## 🔄 Parte 3: Configuração de Portas Trunk

### Comandos Úteis (Para Switch Core - Layer 3)

- `interface <tipo_e_numero>` ou `interface range <tipo_e_intervalo>`
    - **Função:** Entra no modo de configuração de uma porta específica (ex: `GigabitEthernet0/1`) ou de um intervalo de portas (ex: `FastEthernet0/1 - 7`).
- `switchport trunk encapsulation dot1q`
    - **Função:** Define o protocolo de encapsulamento que será usado para "etiquetar" o tráfego das VLANs. Este comando é **obrigatório em switches Layer 3**, como o 3560, antes de ativar o modo trunk.
- `switchport mode trunk`
    - **Função:** Ativa o modo de operação *trunk* na porta selecionada.

### Comandos Úteis (Para Switches de Acesso - Layer 2)

- Nos switches de acesso (modelo 2960), geralmente o comando `switchport trunk encapsulation dot1q` não é necessário, pois eles só suportam este protocolo. Você pode usar diretamente o `switchport mode trunk`.

### Comando de Verificação

- `show interfaces trunk`
    - **Função:** Mostra uma lista de todas as portas que estão operando em modo trunk, seu status e quais VLANs estão autorizadas a passar por elas. Essencial para verificar se suas "autoestradas" estão funcionando.

---

## 💻 Parte 4: Configuração de Portas de Acesso

### Comandos Úteis

- `switchport mode access`
    - **Função:** Configura a porta para operar em modo de acesso.
- `switchport access vlan <ID>`
    - **Função:** Associa a porta de acesso à VLAN específica que você indicar. Todo o tráfego que entrar ou sair por esta porta pertencerá a essa VLAN.

## 🌐 Parte 5: Configuração de DHCP e Roteamento no Roteador

O objetivo é fazer com que o roteador (`R-CIN`) gerencie a comunicação entre as diferentes VLANs (roteamento Inter-VLAN) e distribua endereços IP automaticamente para os PCs (serviço DHCP).

### Etapa A: Criar os Gateways (Router-on-a-Stick)

O roteador precisa de uma "porta" virtual para cada VLAN. Essas são as sub-interfaces.

- `interface <tipo_e_numero>.<ID_DA_VLAN>`
    - **Função:** Cria uma sub-interface virtual. O número após o ponto deve corresponder ao ID da VLAN para manter a organização (ex: `interface GigabitEthernet0/0.10` para a VLAN 10).
- `encapsulation dot1Q <ID_DA_VLAN>`
    - **Função:** Vincula a sub-interface à VLAN correspondente. É assim que o roteador sabe qual tráfego pertence a qual sub-interface.
- `ip address <IP_DO_GATEWAY> <MASCARA>`
    - **Função:** Atribui o endereço IP do gateway para aquela sub-rede/VLAN. Este é o endereço que os PCs usarão para se comunicar com outras redes.

### Etapa B: Configurar o Serviço DHCP

- `ip dhcp excluded-address <IP_A_EXCLUIR>`
    - **Função:** Impede que o servidor DHCP distribua um endereço específico. Use isso para reservar os endereços dos gateways que você configurou na Etapa A.
- `ip dhcp pool <NOME_DO_POOL>`
    - **Função:** Cria um "conjunto" de configurações DHCP com um nome descritivo (ex: `POOL_LAB1`).
- `network <ENDERECO_DA_REDE> <MASCARA>`
    - **Função:** Define a faixa de endereços IP que este pool pode distribuir.
- `default-router <IP_DO_GATEWAY>`
    - **Função:** Informa aos clientes DHCP qual é o endereço do gateway padrão da sua rede.
- `dns-server <IP_DO_DNS>`
    - **Função:** Informa aos clientes qual servidor DNS eles devem usar. Conforme solicitado, utilize o endereço **`8.8.8.8`** (DNS público do Google).

Boa sorte com a configuração!

## 🔌 Conceitos Essenciais: Portas de Acesso vs. Trunk

### Portas de Acesso (Access Ports)

- Pertencem a **uma única VLAN**
- Conectam **dispositivos finais** (PCs, servidores)
- Tráfego **não-etiquetado** (untagged)
- **No projeto:** Portas onde os PCs dos laboratórios se conectam

**Exemplo:** Um PC conectado à porta Fa0/5 de um switch configurado para VLAN 10 só enxerga a VLAN 10.

### Portas Trunk (Trunk Ports)

- Transportam tráfego de **múltiplas VLANs** simultaneamente
- Conectam **switches a switches** ou **switches a roteadores**
- Tráfego **etiquetado** usando protocolo **IEEE 802.1Q**
- **No projeto:** Conexões entre switches de acesso e switch core, e entre switch core e roteador

**Exemplo:** A conexão entre Switch-Acesso-Lab1 e Switch-Core transporta tráfego das VLANs 10, 20, 30, 40 e 50.

## 📖 Fundamentos Teóricos

### O que são VLANs?

**VLAN (Virtual Local Area Network)** é uma tecnologia que permite criar redes lógicas separadas dentro da mesma infraestrutura física.

**Analogia:** Imagine um prédio com vários apartamentos. Embora compartilhem a mesma estrutura física, cada apartamento é privado e isolado. VLANs funcionam assim na rede!

**Benefícios das VLANs:**

- 🔒 **Segurança:** Isola tráfego entre departamentos/grupos
- ⚡ **Performance:** Reduz domínios de broadcast (menos tráfego desnecessário)
- 🎯 **Organização:** Agrupa logicamente dispositivos independente da localização física
- 💰 **Economia:** Não precisa de switches separados para cada grupo

**No projeto do CIn:**

- Cada laboratório será uma VLAN separada
- Lab 1 → VLAN 10
- Lab 2 → VLAN 20
- Lab 3 → VLAN 30
- Lab 4 → VLAN 40
- Lab 5 → VLAN 50

### O que é DHCP?

**DHCP (Dynamic Host Configuration Protocol)** automatiza a configuração de rede dos dispositivos.

**Sem DHCP:** 😫

- Configurar manualmente IP, máscara, gateway, DNS em cada PC
- Alto risco de erros e conflitos de IP
- Muito trabalho em redes grandes

**Com DHCP:** 😊

- PC liga → solicita configuração
- Servidor DHCP responde com todas as informações
- Configuração automática e sem erros

**Funcionamento (processo DORA):**

1. **D**iscover: PC envia broadcast "Preciso de um IP!"
2. **O**ffer: Servidor DHCP oferece um IP disponível
3. **R**equest: PC aceita a oferta
4. **A**ck: Servidor confirma e reserva o IP

### Relação entre VLANs, Sub-redes e DHCP

É crucial entender como estes três conceitos se conectam:

1. **VLAN** (Camada 2 - Link): Cria isolamento lógico usando tags em quadros Ethernet
2. **Sub-rede IP** (Camada 3 - Rede): Define o espaço de endereçamento (calculado na E2)
3. **Gateway** (Camada 3): Porta de saída da VLAN/sub-rede para outras redes
4. **DHCP**: Serviço que automatiza a entrega das configurações (IP, máscara, gateway)

**Exemplo integrado:**

- PC está na VLAN 10 (tag 802.1Q)
- VLAN 10 usa a sub-rede 172.20.X.0/24
- Gateway da VLAN 10: 172.20.X.1
- Pool DHCP para VLAN 10: 172.20.X.2 até 172.20.X.62
- PC recebe automaticamente: IP do pool, máscara /24, gateway .1

## Boas práticas e Erros Comuns

**❌ Erro 1:** VLAN não aparece no switch de acesso

- **Solução:** Certifique-se de criar a VLAN em TODOS os switches

**❌ Erro 2:** Trunk não funciona

- **Solução:** Verifique se ambas as pontas da conexão estão configuradas como trunk

**❌ Erro 3:** Esqueceu de excluir o IP de gateway

- **Solução:** O gateway não pode ser distribuído! Sempre exclua antes de criar o pool

**✅ Dica:** Use nomes descritivos para VLANs - ajuda na documentação e troubleshooting

**✅ Dica:** Documente todas as portas trunk e access - você precisará disso na apresentação final

**✅ Dica:** O comando `interface range` economiza tempo ao configurar múltiplas portas

**Próxima etapa:** Com VLANs e DHCP configurados, você implementará o roteamento Inter-VLAN para permitir comunicação entre os laboratórios (E4).
