# E3 - Design de Segmentacao (VLANs e DHCP)

**Grupo 1**

## 1. Decisões de Projeto

A segmentação segue diretamente o plano VLSM da E2: uma VLAN por laboratório, com os dois laboratórios maiores divididos em dois switches de acesso, mas mantendo a mesma VLAN em ambos os switches.

Para esta etapa, a estratégia adotada foi:

- **DHCP centralizado no roteador `R-CIN`**
- **Roteamento Inter-VLAN por Router-on-a-Stick**
- **Uma VLAN por laboratório:** `10`, `20`, `30`, `40` e `50`
- **Gateway como primeiro IP útil de cada sub-rede**
- **DNS entregue via DHCP:** `8.8.8.8`

A implementação no arquivo Packet Tracer reutiliza a topologia física já existente em `deliverables/topologia.pkt`, em que:

- `R-CIN Gi0/0` conecta ao `SW-CORE Fa0/1`
- `SW-CORE Fa0/2-Fa0/8` conectam aos sete switches de acesso
- em cada switch de acesso, a porta uplink é a `Fa0/1`

## 4. Estratégia DHCP

O roteador `R-CIN` concentra o serviço DHCP para simplificar a administração e manter a correlação direta entre:

- subinterface `Gi0/0.<VLAN>`
- gateway da VLAN
- pool DHCP correspondente

Os endereços dos gateways foram excluídos da distribuição dinâmica:

- `172.20.1.1`
- `172.20.1.65`
- `172.20.1.129`
- `172.20.1.161`
- `172.20.1.193`

Os pools usados no roteador são:

- `LAB1_VLAN10`
- `LAB2_VLAN20`
- `LAB3_VLAN30`
- `LAB4_VLAN40`
- `LAB5_VLAN50`

## 5. Comandos-Chave da Implementação

### 5.1 Roteador `R-CIN`

```text
enable
configure terminal
hostname R-CIN

interface GigabitEthernet0/0
 no ip address
 no shutdown

interface GigabitEthernet0/0.10
 encapsulation dot1Q 10
 ip address 172.20.1.1 255.255.255.192

interface GigabitEthernet0/0.20
 encapsulation dot1Q 20
 ip address 172.20.1.65 255.255.255.192

interface GigabitEthernet0/0.30
 encapsulation dot1Q 30
 ip address 172.20.1.129 255.255.255.224

interface GigabitEthernet0/0.40
 encapsulation dot1Q 40
 ip address 172.20.1.161 255.255.255.224

interface GigabitEthernet0/0.50
 encapsulation dot1Q 50
 ip address 172.20.1.193 255.255.255.224

ip dhcp excluded-address 172.20.1.1
ip dhcp excluded-address 172.20.1.65
ip dhcp excluded-address 172.20.1.129
ip dhcp excluded-address 172.20.1.161
ip dhcp excluded-address 172.20.1.193

ip dhcp pool LAB1_VLAN10
 network 172.20.1.0 255.255.255.192
 default-router 172.20.1.1
 dns-server 8.8.8.8

ip dhcp pool LAB2_VLAN20
 network 172.20.1.64 255.255.255.192
 default-router 172.20.1.65
 dns-server 8.8.8.8

ip dhcp pool LAB3_VLAN30
 network 172.20.1.128 255.255.255.224
 default-router 172.20.1.129
 dns-server 8.8.8.8

ip dhcp pool LAB4_VLAN40
 network 172.20.1.160 255.255.255.224
 default-router 172.20.1.161
 dns-server 8.8.8.8

ip dhcp pool LAB5_VLAN50
 network 172.20.1.192 255.255.255.224
 default-router 172.20.1.193
 dns-server 8.8.8.8
```

### 5.2 Switch Core `SW-CORE`

```text
enable
configure terminal
hostname SW-CORE

vlan 10
 name LAB1
vlan 20
 name LAB2
vlan 30
 name LAB3
vlan 40
 name LAB4
vlan 50
 name LAB5

interface range FastEthernet0/1 - 8
 switchport trunk encapsulation dot1q
 switchport mode trunk
```

### 5.3 Switches de Acesso

Exemplo para os switches do Lab 1:

```text
enable
configure terminal
hostname SW-LAB35-A

vlan 10
 name LAB1

interface FastEthernet0/1
 switchport mode trunk

interface range FastEthernet0/2 - 19
 switchport mode access
 switchport access vlan 10
```

Exemplo para um switch com laboratório dedicado:

```text
enable
configure terminal
hostname SW-LAB20-A

vlan 30
 name LAB3

interface FastEthernet0/1
 switchport mode trunk

interface range FastEthernet0/2 - 21
 switchport mode access
 switchport access vlan 30
```

## 7. Observações de Funcionalidade

- A lógica de segmentação está pronta para operar com DHCP automático por VLAN.
- Os laboratórios de 35 e 30 hosts compartilham a mesma VLAN em dois switches físicos, preservando a exigência de capacidade da E1 sem criar sub-redes extras.
- Como o arquivo `topologia.pkt` parte de uma topologia sem a população completa de PCs, a validação final por concessão DHCP em todos os laboratórios depende da inserção/ativação dos hosts finais nas portas de acesso correspondentes.

## 8. Resumo

Esta entrega fecha a etapa de segmentação com:

- cinco VLANs coerentes com o VLSM da E2;
- trunks entre roteador, core e acesso;
- portas de acesso mapeadas por laboratório;
- DHCP centralizado e documentado no `R-CIN`.

Com isso, a próxima etapa (E4) pode se concentrar nas evidências de conectividade e na rota de saída para a RNP.
